🚀 HOW TO RUN COGNIPLAN (PRODUCTIVITY TRACKER)
Location: D:\Projects\Tracker
OPTION 1: THE EASY WAY (AUTOMATIC)
Double-click the START_APP.bat file in the root folder.
This script is configured to:

Activate the Python Virtual Environment (.venv).

Start the FastAPI backend server.

Launch your browser automatically to http://127.0.0.1:8000.

OPTION 2: THE MANUAL WAY (VS CODE / POWERSHELL)
If you prefer running it manually to see the logs and debug, follow these steps:

Open Terminal in the Tracker folder.

Activate the Environment:

PowerShell
.\.venv\Scripts\Activate.ps1
Enter the Backend Directory:

PowerShell
cd backend
Start the Server:

PowerShell
uvicorn main:app --reload
View the App: Open your browser and go to http://127.0.0.1:8000.

📂 IMPORTANT: FOLDER STRUCTURE
To ensure the app runs correctly, keep the files organized as follows:

/backend: Contains main.py, models.py, and your .env (connected to Supabase).

/docs: Contains your frontend (index.html, style.css, app.js).

Note: We use the docs folder instead of frontend so the app works both locally and on GitHub Pages.

☁️ DATABASE STATUS
Provider: Supabase (PostgreSQL)

Connection: Managed via the DATABASE_URL in your backend/.env.

Auto-Sync: Tables are automatically checked/created every time the backend starts.

🛠️ TROUBLESHOOTING
"ModuleNotFoundError": If you see a missing module (like psycopg2), run pip install -r backend/requirements.txt.

"Port 8000 in use": If the server won't start, close any other Python windows or run uvicorn main:app --reload --port 8001.

🌍 HOW TO RUN LIVE HOSTED (PRODUCTION)
This project is configured to be hosted live using GitHub Pages (frontend) and Render (backend).

To make your changes live for the world, follow these steps:
1. Open docs/app.js.
2. Locate the "CORE API FETCHER" section (around line 37).
3. Comment out the localhost URL by adding `//` before it:
   // const API_BASE_URL = "http://localhost:8000";
4. Uncomment the onrender.com URL by removing the `//`:
   const API_BASE_URL = 'https://cogniplan-siaf.onrender.com';
5. Save the file.
6. Push to GitHub:
   git add .
   git commit -m "Switch to production URL"
   git push origin main
7. Your app will be live on your GitHub Pages URL, and it will communicate with the live Render backend.

💻 HOW TO RUN LOCALLY (DEVELOPMENT)
Whenever you want to build a new feature or test a change locally on your machine:

1. Open docs/app.js.
2. Comment out the production URL:
   // const API_BASE_URL = 'https://cogniplan-siaf.onrender.com';
3. Uncomment the localhost URL:
   const API_BASE_URL = "http://localhost:8000";
4. Save the file.
5. Start the backend: Open terminal, navigate to the tracker folder, and run:
   .\START_APP.bat
   (Or manually: cd backend -> .\.venv\Scripts\Activate.ps1 -> uvicorn main:app --reload)
6. Open docs/index.html with VS Code Live Server.
7. Start coding!

☁️ THE SHARED DATABASE MAGIC
Both your local laptop and your live Render server point to the exact same Supabase database in the cloud. Data isn't saved on your laptop or on Render—it's saved in Supabase. So, no matter where you access the backend from, you are looking at the exact same data.

Stop the server: Go to your terminal and press CTRL + C to stop FastAPI.

Delete the database: Go to your VS Code file explorer on the left side, right-click the productivity.db file, and hit Delete. (Don't worry, your SQLAlchemy code will automatically rebuild the empty tables the next time it runs).

Run the seed script: In your terminal, make sure you are in the main Tracker folder (not inside the backend folder, because your code says from backend.database...), and run this exact command:

Bash
python -m backend.seed
The Result: The terminal will print: ✅ Seeded 5 habits and 5 todos successfully.

Start the server again: Run your uvicorn command, refresh your browser, and the frontend will instantly have 30 days of data.
